Working with Javascript

And It's differences from Ruby.

March 22nd, 2015

There were two main differences between Javascript and Ruby that really mattered to me, and fortunately they are both rather easy to explain. Javascript requires strict syntax but is permissive about variable operations, while Ruby does not, and Ruby lets you link multiple files together, where Javascript does not.

So, I actually like the JavaScript syntax more for most things, but that may be because I'm used to it. I find using brackets instead of the ruby 'end' keywords are very helpful, and much faster to type. Javascript also requires a semicolon at the end of statements, which can be a bit annoying sometimes. I think Ruby really is much better in that area. Another area Ruby is preferred is that is has methods for doing things "in place," which JS lacks. I found myself making many more assignments than I would normally do in Ruby. Finally, Javascript coerces types when you make a non-nonsensical operation, for example, comparing if 0 == false. Now, clearly a number does not equal a boolean, but javascript will attempt to make the statement evaluate anyway, so it assumes that 0 is being used to represent the boolean of false, and therefore returns that 0 does in fact equal false. Doing this silently in the background can lead to headaches, which is why, while it may sound user friendly, I actually prefer Ruby in this respect, because it doesn't coerce data types.

This gripe is relevant to the solo challenge for this week, where I really would have benefited from being able to use Ruby's file linking. Basically, there is no 'require' equivalent in javascript. Sure, HTML pages can load multiple javascript files, but I wasn't making an HTML page, instead focusing my efforts into making an ascii game that could be run in the terminal. This meant that all of my functions and other code sections that would normally be put into separate files all had to be crammed into a single document. This made navigating and making changes an extreme chore. everything else about javascript I could work around easily, but I really think ruby wins out in this category. I want to be able to break things down into multiple files for a project as complex as this, even if it's for readability only.
